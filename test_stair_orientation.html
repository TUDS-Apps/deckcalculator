Stair Orientation Debug Test

Test Instructions:

  1. Open the main deck calculator application in another tab
  2. Draw a deck shape and generate a plan
  3. Click "Add Stairs" and place stairs on a 5'3" rim joist section
  4. Watch the browser console (F12) for debug output
  5. Note the stair orientation information printed to console

What This Test Does:

We've added debug logging to the stair drawing code that will output:

  • Rim direction: The direction vector of the rim joist edge
  • Initial perpendicular: The initial perpendicular vector (before orientation correction)
  • Test points: Two test points on either side of the rim joist
  • Inside/Outside status: Whether each test point is inside or outside the deck polygon
  • Final direction: The final perpendicular vector used for stair placement

Expected Behavior:

Stairs should always point away from the deck interior. The debug output will show which test point is outside the deck, and stairs should point in that direction.

Common Issues:

  • Wrong initial perpendicular: The perpendicular calculation might be reversed
  • Inconsistent rim joist direction: Rim joist p1â†’p2 might not be consistent
  • Point-in-polygon errors: The test might incorrectly identify inside/outside
Debug output will appear in the browser console when you place stairs.

To open the console:
- Chrome/Edge: Press F12 or Ctrl+Shift+I (Cmd+Option+I on Mac)
- Firefox: Press F12 or Ctrl+Shift+K (Cmd+Option+K on Mac)
- Safari: Enable Developer menu in Preferences, then Cmd+Option+C
